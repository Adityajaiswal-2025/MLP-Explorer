1. Representation Power of Multilayer Perceptrons

A Multilayer Perceptron (MLP) is a class of feedforward artificial neural network. It consists of at least three layers of nodes: an input layer, a hidden layer, and an output layer. Except for the input nodes, each node is a neuron that uses a nonlinear activation function.

Mathematical Neuron Model

y = σ( ∑ wi xi + b )

Where x represents inputs, w represents weights, b is bias, and σ is the activation function.

What makes MLPs exceptionally powerful is their ability to classify data that is not linearly separable. A single-layer perceptron can only draw a straight line (or plane) to separate two classes, failing on simple relationships like XOR. By nesting multiple layers together, an MLP can perform coordinate transformations, warping space so that complex curves are flattened out and become linearly separable by the final layer.

2. Bending Space: The Role of Hidden Layers

To understand hidden layers, think of them as performing a series of coordinate transformations. When a dataset (like XOR or a spiral) is passed through a hidden layer, the layer maps the 2D input coordinates onto a new, high-dimensional space defined by the activations of the hidden neurons.

Interactive Feature space Insight

Look at the Feature Learning (Hidden Space) plot in our playground! For the XOR problem:

  • In the input space (2D), Quadrants I & III belong to class 1, while Quadrants II & IV belong to class 0. A straight line cannot separate them.
  • After passing through a hidden layer with 2 neurons, the four clusters are warped. The activation values map them to a new 2D grid where class 1 clusters move to one side, and class 0 clusters move to the other.
  • In this new space, the classes can be separated by a single straight line, which the output layer easily fits!
Adding more layers allows the network to build a hierarchy of features. Earlier layers represent simple local details (like lines and edges), while deeper layers combine them into global shapes and complex manifolds.

3. The Spark of Non-Linearity: Activation Functions

Without activation functions, a multilayer neural network collapses into a simple linear model. Let's look at the math: if all layers are linear, we have:

y = W2 ( W1 x + b1 ) + b2 = (W2W1) x + (W2b1 + b2) = W' x + b'

No matter how many hidden layers are added, the composition of linear transformations is always just another single linear transformation. Therefore, the network cannot learn non-linear decision boundaries.

Common Activation Functions:

ReLU (Rectified Linear Unit)

f(x) = max(0, x)

The standard choice for modern deep networks. It is computationally efficient and mitigates the vanishing gradient problem. However, it can suffer from "dying ReLU" if neurons receive gradients that push weights to always output zero.

Sigmoid

f(x) = 1 / (1 + e-x)

Outputs values between 0 and 1, representing a probability. Often used in the output layer for binary classification. Can cause "vanishing gradients" during training since its derivative is close to zero for large values.

Tanh (Hyperbolic Tangent)

f(x) = (ex - e-x) / (ex + e-x)

Outputs values between -1 and 1. Zero-centered, which makes training slightly easier than Sigmoid, but still suffers from vanishing gradients when neurons saturate.

4. Universal Approximation Theorem

The Universal Approximation Theorem (proven by George Cybenko in 1989 for Sigmoid activations, and later generalized by Kurt Hornik in 1991) is a fundamental pillar of Deep Learning. It states:

"A feedforward neural network with a single hidden layer containing a finite number of neurons can approximate any continuous function on compact subsets of &mathbb;Rn to arbitrary accuracy, provided it uses a non-constant, bounded, and continuous activation function."

In simple terms, this means that even a simple 3-layer network (1 hidden layer) can theoretically learn any pattern, no matter how complex, given enough neurons. However, the theorem doesn't guarantee that the model is learnable (optimizable with standard algorithms like Gradient Descent) or that a single wide layer is efficient. In practice, **deeper** networks (adding more hidden layers rather than just widening one) learn hierarchical features and generalize far better with fewer total parameters.

5. Network Capacity: Underfitting vs. Overfitting

The capacity of an MLP is determined by its architecture (number of layers and neurons per layer) and the duration of training. Finding the right balance is key to creating model representations that generalize to unseen data.

Underfitting

  • Cause: Network is too simple (e.g. 1 layer with 1 or 2 neurons) or trained for too few epochs.
  • Sign: High training loss and low accuracy. The decision boundary is a straight line or very simple curve, failing to split the datasets (like XOR or Spiral).

Overfitting

  • Cause: Network is overly complex (e.g. 5 layers with 10 neurons each) or trained for too long on noisy datasets.
  • Sign: Very low training loss but fails to generalize. The decision boundary becomes jagged, wrapping closely around individual noise points to fit them, rather than capturing the global trend.
